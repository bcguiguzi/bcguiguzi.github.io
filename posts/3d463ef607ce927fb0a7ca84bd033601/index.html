Windows 7：故障排除连接

08/17/2016

本文内容

这一简单的网络连接故障排除练习能够让您在日后遇到类似问题时扩展思维、举一反三。

William R. Stanek

Windows 7 的内置故障排除功能可以帮助您快速解决大多数问题。 然而，你偶尔会遇到其他不能快速或很容易解决的问题。

考虑以下情况：有一个系统，是一个相当复杂的小型企业网络上的许多运行。 网络具有单个路由器和多个网络交换机。 当操作系统启动时，计算机不能连接到网络或互联网。 计算机运行 Windows 7 x 64 并具有单一 1.0Gbps 网络适配器。

任何连接性问题的故障排除的第一步总是相同。 请检查网络状态。 这很容易做。 只是看至系统托盘。 在这里，系统托盘中的网络图标显示的警告标志。 单击网络图标，然后单击打开网络和共享中心透露此计算机与单个网络适配器认为它是多个网络上。

怎么了这是在启动时，计算机检查网络配置，发现适配器，但无法确定究竟如何它连接到网络。 其结果是，计算机有没有网络连接，并没有任何的互联网连接。 关于这一问题在这里是有什么想法？ 这里有一些事情要考虑：

这是操作系统问题吗？

这是一个硬件问题吗？

这是配置有问题吗？

这只是一件奇事或临时呃逆么？

你将怎样有关故障诊断和解决问题？ 让我们更深入地挖掘，看一看。 这种类型不是问题的这么多问题，这是问题的一个有待解决的难题。 所以从何处开始解开这一难题呢？

让我们开始用"检查这一明显"并调用该步骤 1。 我们正在设法回答的问题是这是否只是一件奇事或临时呃逆。 在这里还暗示是基础知识的检查。 这意味着，检查电缆，确保它们连接在合适的位置。

步骤 1：检查明显

所以这是一件奇事、 信号呃逆或基本电缆问题吗？ 这是很简单，以及确定。 如果这是到交换机之间的连接有问题，在计算机接通和路由器，或路由器与互联网，问题会显示在网络和共享中心为缺乏网络和互联网之间的连接。

单击摘要网络图中的红色 X 启动内置网络疑难解答。 故障诊断程序将尝试连接到 Microsoft 来解决您的 Internet 连接。 交换机和路由器之间断开连接时，它是在此时很大程度上无法区分 TCP/IP 配置问题。

当您继续执行自动故障排除时，网络疑难解答大多是检查配置问题，并不一定能够告诉你有一个断开交换机和路由器之间。 疑难解答按此顺序执行以下检查：

网络连通性

名称解析

远程主机可用性

网络配置

网络网关可访问性

重新检查网络配置

若要查看自动故障排除的详细的结果，您需要单击查看详细信息链接它将提供。 然后，您将能够访问疑难解答报告。 如果这是有问题的计算机连接到交换机，问题将显示一个红色的 X，在特定的计算机和互联网之间的网络和共享中心。

再次单击摘要网络图中的红色 X 启动内置网络疑难解答。 疑难解答将快速将告诉您可以将以太网电缆插入计算机，然后单击检查一下如果固定的问题。

当然，您可能已检查的网络连接窗口中的问题，你会知道同样的事情。 局域网连接失败，因为电缆被拔出。

是否计算机端或开关结束已断开连接，它只需要 Windows 7 几秒钟来检测您已经重新连接的以太网电缆。 然后，Windows 7 将尝试将您连接到局域网。 如果计算机已连接到交换机并不向其余的局域网和互联网，可能的 Windows 7 将显示您连接到未识别的网络，你有没有 Internet 访问权。

但是，计算机在这里，遇到的问题是非常不同的。 计算机可能认为这是对多个网络。 它有没有网络连接和无互联网连接，即使基本布线好。 因此是这只是一个奇怪现象或呃逆吗？ 检查的一种方法是只禁用然后再启用的网络适配器。

这将强制重新计算的网络组件和配置的计算机。 更重要的在这宗个案，问题似乎消失。 网络连接返回，一切看起来很好。 但是真正解决问题吗？ 不在这种情况下。 所以作为这方面的问题不是一件奇事或呃逆，让我们转至步骤 2。

步骤 2：检查硬件

在这里，您想要回答的是这是一个基本的硬件问题。 您可能已经回答这如果你看过网络连接页。 如果活动的网络适配器有不利的地位，例如残疾人士或网络电缆拔出，它将显示这和纠正行动很容易。 您启用已禁用的适配器，或将电缆插入一个包含插入的电缆。

怎么样网络连接页面为空或否则不会有局域网连接，它应该吗？ 这不会发生。 您应然后单击控制面板 |网络和互联网 |网络和共享中心 |更改适配器设置，只是为了查找列出，没有本地连接，即使你知道计算机有一个有线的网络适配器。

在这种情况下，快速检查设备管理器会告诉你什么。 可从计算机管理访问设备管理器。 要启动计算机管理，单击开始，在搜索框中键入 compmgmt.msc，然后按 Enter。

在计算机管理中选择设备管理器，然后展开网络适配器节点，以检查已安装的适配器的状态。 如果您看到一个警告图标，该设备将具有错误状态，您可以通过右键单击相关的条目，然后选择属性检查。

如果该设备不根本，将会发生如果卸载该设备，您可以右键单击列出网络适配器节点并选择扫描硬件改动，让 Windows 检查可用但未安装设备。 请考虑下列事项：

如果设备已列出，但有错误状态，您可以尝试通过右键单击设备并选择更新驱动程序软件安装相关的驱动程序软件。 接下来，单击搜索自动更新驱动程序软件。 如果可用更新的软件，您将能再安装它。

如果 Windows 确定是最新的驱动程序软件，但该设备仍然不工作，你就会想要访问设备制造商的 Web 站点、 查找当前的驱动程序，然后手动安装驱动程序。 这样做应解决设备问题。

设备的正常状态可能不会有基本硬件问题。 若要解决此类问题，请查找测试机。 转到设备管理器中，展开网络适配器节点，右键单击适配器，然后选择卸载。 在测试计算机上卸载该适配器后，可以通过此故障排除方案中的步骤 2 工作并查看是否您可以使用提供的信息以解决问题。

显然，一些类型的问题走上阻止计算机正确连接到网络和互联网。 在步骤 1 中，你签了显而易见的事实。 在步骤 2 中，你已检查硬件。 如果您是经验丰富，在这种事情，所有这些相关检查大概花了一分钟或更少。

步骤 3：连接检查

通常情况下，这种类型的问题很容易解决。 不过，此计算机与单个网络适配器认为它是对多个网络。 问题的症结所在。 怎么了这是在启动时，计算机检查网络配置，发现适配器，但无法确定究竟如何它连接到网络。 其结果是，计算机有没有网络连接，并没有任何的互联网连接。

在步骤 3 中，您将尝试以确定是否存在配置问题。 通常情况下，检查配置问题的最快方法是使用命令提示符。 显示网络配置，通过输入 ipconfig/所有。 在输出中，检查 IPv4 (并在适当时 IPv6) 配置。 对于 IPv4，看具体看下面：

分配的值应适合于计算机所连接到的网络。 如果启用动态主机配置协议 (DHCP)，则请确保计算机已连接到相应的 DHCP 服务器。

如果计算机未连接到 DHCP 服务器，它将自动的专用 IP 地址 (APIPA) 范围中具有一个 IP 地址：169.254.0.1 到 169.254.255.254。 APIPA 可能具有的计算机无法适当地在网络上进行通信。 其结果是，Windows 定期检查变得可用的 DHCP 服务器。

如果计算机看起来有相应的网络设置，请尝试 ping 或 tracert 到默认网关。 在某些情况下，ping 阻止的防火墙设置，但 tracert 应工作。 如果 ping 或 tracert 失败在这里，它很可能指示计算机和默认网关之间的连接问题或默认网关本身有问题。 若要解决连接问题，将检查电缆连接和计算机和默认网关之间切换。

要解决一个网关问题，需要看看本身来确定其状态和配置的设备。 请记住，但是，您不能连接到一个不存在的网关。 关于网络 192.168.27.x 配置的计算机应在大多数情况下有 192.168.27.1 的默认网关。 然而，如果网络设置是错误的计算机实际上是在执行 ping 命令 192.168.27.1 192.168.12.x 网络，它可能不会返回任何结果因为正确的网关的 IP 地址是最有可能的 192.168.12.1。

如果计算机可以访问的默认网关，请尝试 ping 或 tracert 已知的互联网网站。 再次，在某些情况下，防火墙可能会阻止 ping，但 tracert 应工作。 Ping 或在这里 tracert 故障将表明互联网连接或互联网服务的问题。 您将需要确保切割更多一点。 例如，如果您无法 ping 或通过站点名称如 tracert www.yahoo.com tracert 您应该确保这不是 DNS 问题。

要检查此项，您应该 ping 或 tracert 外部站点由其 IP 地址。 如果你能到外部站点的 IP 地址)，但其完全限定的名称，您可能有一个 DNS 问题。 通过验证 DNS 服务器的 IP 地址和 DNS 服务器 IP 地址的服务器设置适当解决 DNS 问题。 您还可能需要刷新 DNS 解析器缓存。 要解决网络连接问题，请检查内部路由器与路由到互联网的设备之间的电缆连接。 此外应检查设备状态。

在此方案中，计算机具有 DHCP 分配的 IP 设置，并且能够从 DHCP 服务器获取这些设置。 因此，检查 TCP/IP 配置在命令行上的似乎提供了正确的设置。

如果你认为问题有关的 DHCP 设置，您可以执行各种疑难解答 Ipconfig 命令行任务。 Ipconfig /release Ipconfig / 续订，Ipconfig/flushdns 和 Ipconfig /registerdns 是非常方便的所有命令。

如果你认为问题与指定的设置，您可以通过单击控制面板查看适配器设置 |网络和互联网 |网络和共享中心 |更改适配器设置。

然后在网络连接页中，将用鼠标右键单击相应的适配器，然后选择属性。 在属性对话框中，选择 Internet 协议版本 4，单击属性。 现在，您可以查看的 IPv4 配置。

在这一阶段解决网络问题的一种方法是手动分配您知道应该工作的 IP 设置。 如果你能够手动分配设置并连接到网络和互联网，则问题很可能被有关 DHCP。 以下是可能的嫌疑人：

在网络上的另一台计算机可能或已经在计算机尝试使用相同的 IP 地址。

不正确的网关设置可能无法在计算机到正确的 DHCP 服务器获取。

DHCP 服务器本身可能有问题，可能指定了不适当的 IP 设置。

任何猜测，手动分配 IP 设置时，会发生什么？ 嗯，它不会强迫要修改其配置的计算机。 更重要的是，在此特定情况下，问题似乎消失。 网络连接返回，一切看起来很好。 一切真正好的不过是吗？ 只有你就会知道。
