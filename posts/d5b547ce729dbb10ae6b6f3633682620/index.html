那些劝退小白的指针，其实并不难

说到指针，真是让人又爱又恨，爱它是因为指针真的可以为所欲为，恨它是因为对于新手来说，一不小心就晕了。像什么常量指针、指针常量、指针数组、函数指针、二级指针、智能指针。。。。。。其实这么多奇奇怪怪的指针，不管它的语法有多么花里胡哨，本质上都是一个地址。
举个栗子
一、普通指针

	int age = 18;
	int* p = &age;
	cout << "age的地址：" << &age << endl;
	cout << "age的地址：" << p << endl;
	cout << "age=" << age << endl;
	cout << "age=" << *p << endl;

这时候p就是变量age的地址，那么p就是这个地址上的值，也就是18，可以给p赋一个新值，然后age的值也会变。但是age如果是一个常量，就不能这么干了，因为*p可以改变age的值，但是age又不能改，这就产生矛盾了，编译器会报错。
所以普通指针不能指向常量。

二、常量指针

	int age = 18;
	const int* p = &age;
	cout << "age的地址：" << &age << endl;
	cout << "age的地址：" << p << endl;
	cout << "age=" << age << endl;
	cout << "age=" << *p << endl;

const离谁近，就修饰谁，所以这里修饰的是int，也就是说p是不能改的，就算它指向的是一个变量，也不能通过p去修改这个变量，指向常量就更没问题了。这个p也可以指向其他常量或变量。

三、指针常量

	int age = 18;
	int* const p = &age;
	cout << "age的地址：" << &age << endl;
	cout << "age的地址：" << p << endl;
	cout << "age=" << age << endl;
	cout << "age=" << *p << endl;

它跟前面的普通指针差不多，唯一的区别是，它只能指向age，不能指向别的变量了。

再举个栗子
二级指针
大家都知道变量有地址，那指针是不是变量？它是不是也有地址？于是就有了二级指针，它指向一个指针的地址。

	int age = 18;
	int* p = &age;
	int** p1 = &p;
	cout << "age的地址：" << p << endl;
	cout << "p的地址：" << &p << endl;
	cout << "p1：" << p1 << endl;
	
	cout << "age=" << age << endl;
	cout << "*p=" << *p << endl;
	cout << "*p1=" << *p1 << endl;
	cout << "**p1=" << **p1 << endl;

指针p保存的是age的地址，所以p就得到了地址上的值，也就是age。二级指针p1保存的是p的地址。那么p1就得到了地址上的值，也就是p，而p保存的是age的地址，所以再来一次**p，就是age。
还有三级指针、四级指针。。。。。。都是这么玩的，一层一层的剥开，就可以了，但是几乎用不到，常用的就是二级指针。

这就是指针最基本的用法，万变不离其宗，其他指针都是这么玩的。
