git之更新

// 更新Git(需要先进入目录)(服务器不能立刻更新代码时使用)
git reset --hard

// 将本地作为最新的文件强制提交到服务器(当本地文件不能通过(Force Update)的时候使用)
1.将本地项目和服务端master分支进行同步或Fetch
2.在有冲突的文件右键点击进行OverWrite并Mark一下再Pull
3.若还是不行就修改文件进行提交
4.push到服务器
还可以打开GitStaging进行操作
