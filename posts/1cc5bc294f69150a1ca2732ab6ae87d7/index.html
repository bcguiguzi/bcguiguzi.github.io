java里面的CompletableFuture的异常和事务问题和AtomicBoolean解决办法

我们写代码的时候进场会使用到CompletableFuture.runAsync这样的异步操作,大家有没有想过异步里面的内容是否在事务里面并且里面的异常是否能抛出到主线程呢?

    @Autowired
    private MyTableDao myTableDao;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void testAsync() throws InterruptedException {
        CompletableFuture.runAsync(() -> {
            MyTable myTable = new MyTable();
            myTable.setName("测试用");
            myTable.setAge(0);
           myTableDao.updateByAge(myTable);
        });

        MyTable myTable = new MyTable();
        myTable.setName("测试用2");
        myTable.setAge(66);
        myTableDao.updateByAge(myTable);
        Thread.sleep(5000);
        int ii = 3/0;
    }
age 0 修改前name值:123 修改后 测试用
age 66 修改前name值:123 修改后 123

这里为了让异步里面的内容先执行完,在主线程里面睡了5秒,通过结果可以发现异步里面的内容是不会参与到当前事务的

解决办法

这里我们通过引入AtomicBoolean这个适用于多线程安全的类来解决问题

    @Autowired
    private MyTableDao myTableDao;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void testAsync() throws InterruptedException {
        AtomicBoolean atomicBoolean = new AtomicBoolean(false);
        CompletableFuture<Void> asyncReturn = CompletableFuture.runAsync(() -> {
//            try {
                MyTable myTable = new MyTable();
                myTable.setName("测试用");
                myTable.setAge(0);
                myTableDao.updateByAge(myTable);
                int ii = 3 / 0;
//            } catch (Exception e) {
//                atomicBoolean.set(true);
//            }
        });

        Thread.sleep(5000);
        MyTable myTable = new MyTable();
        myTable.setName("测试用2");
        myTable.setAge(66);
        myTableDao.updateByAge(myTable);
        try {
            // 等待异步执行完成获取返回值
            asyncReturn.get();

            throw new RuntimeException();

这里我们通过获取异步的返回值来查看其是否有报错,切记这里异步里面不可以try catch否则这里也获取不到报错,通过修改库里面的数据可以发现CompletableFuture是不参与到当前事务的,其会直接执行成功,参与不到事务的原因是因为事务是存储在本地线程里面的 异步是另起线程导致没办法存在于同一个线程

这里补充一个知识点关于AtomicBoolean

AtomicBoolean这个东西是在多线程里面线程安全的,他因为里面的value字段被volatile修饰
当有一个线程要对这个值进行修改的话他会先去内存里面获取最新的值,修改后也会将该值推送到内存里面保证改数据对其他线程的可见性 我们通过下面代码来演示一下

    public static void main(String[] args) throws Exception{
        AtomicBoolean atomicBoolean = new AtomicBoolean(false);
        ThreadLocal local = new ThreadLocal();
        CompletableFuture.runAsync(() -> {
            try{
                int ii = 3/0;
            }catch (Exception e){
                local.set(true);
                atomicBoolean.set(true);
            }
        });
        Thread.sleep(5000);
        System.out.println(atomicBoolean.get());
        System.out.println(local.get());
    }
结果:true
	 null

通过结果发现,本地线程变量只能对当前线程进行数据共享,而AtomicBoolean可以对所有线程数据进行共享
但是Volatile是没办法保证原子性的,所以我们的Atomic就发挥了神作用,他会通过CAS也就是自旋锁来保证当前的值是最新的,下面的代码简单还原自旋操作

public final int incrementAndGet() {
        for (;;) {
            //获取当前内存里面的值
            int current = get();
            //设置期望值 希望修改的数据
            int next = current + 1;
/*调用Native方法compareAndSet，执行CAS操作这里面的操作就是会拿current
的值和当前主内存里面的值进行对比如果正确就修改当前主内存里面的值是next
不正确就会将旧值进行+1的操作并且预期值也需要进行加加一操作再去和主内里
面的值进行对比,如果对比相同了就将预期值更新到内存里面替换原来的值*/
            if (compareAndSet(current, next))
                //成功后才会返回期望值，否则无线循环
                return next;
        }
    }
