1 计算机网络基础

  1. 三大特征:信息化 /网络化/数字化

  2. 分类1: 计算机网络/电信网络/有线电视网 ;互联网(资源共享/共享)与互连网
    分类2:广域网WAN/城域网Man/局域网LAN/个人局域网PAN
    分类3: 公用网/专用网

  3. 发展历史:arpanet/三级结构互联网(主干网/地区网/校园网)/ISP结构的互联网(主干ISP/地区ISP/本地ISP)

  4. 组成:边缘(C/S或P2P模式通信)/核心部分(电路/报文/分组交换)

  5. 性能:

    1速率:
    1Gb=10^3 Mb=10^6 Kb =10^9bit
    1GB=1024MB=2^20 KB=2^30 Byte=2^30 * 8 bit
    2带宽: 频带宽度/最高数据率
    3吞吐量:通过某个网络(接口等)的实际数据量
    4时延:
    发送时延: 电脑处理, =帧长度bit / 发送速率bit/s
    传播时延: 通过链路时间,=信道长度m / 传播速率 m/s
    处理时延: 路由器处理(查找转发表等)
    排队时延: 经过路由器等待处理
    时延带宽积:传播时延*带宽
    往返时间RTT: 发送时间=数据长度/发送速率(一方主机确认信息的时间)
    利用率: 信道利用率和网络利用率

相关技术

  1. 互联网交换点 IXP: 两个网络直接相连并交换分组,无需通过第三个网络来转发分组

Rip 路由更新操作

路由器6 收到路由器4的更新路由信息, 尝试更新路由器6的路由表
1 r4 中距离都加1
2 r4中的信息在r6中没有,添加
3 下一跳路由器相同: 若距离变大或小,则更新
下一跳路由器不同, 比较距离, 若小于r6表中距离则更新,否则不更新

ARP协议:

解决的问题: 同一局域网中, IP 到 Mac地址的解析(I儿子 找 M妈妈; 高层的IP 找 低层的Mac )

要点:  在主机ARP 高速缓存中存放IP 到 Mac的映射表( 动态更新)
过程: 
先在本地缓存表中查找,有则将该Mac地址写入帧并转发; 
没有,广播自己IP/Mac 以及要找对方主机的IP地址, 运行arp进程的都会收到;  
某主机收到该IP地址与自己IP地址相符则单播给对方; 且发送方记录映射关系

数据链路层

功能:

  1. 两台相邻主机间的数据传输

  2. 数据传输中的差错控制

  3. 封装成帧
    数据前后添加首部和尾部—帧
    帧定界: 确定帧的界限
    帧开始符:soh
    帧结束符:eot

  4. 透明传输

    无论什么比特的数据,都能没有差错的通过数据链路层
    数据中出现soh或eot, 在前加入转义字符esc
    如数据中的eot变为esceot

  5. 差错检测
    传输过程不理想,有比特差错,进行检测
    方法:

循环冗余校验CRC

  1. 前提条件: 二进制 与 K(x)多项式的转换 / 约定好的除数 K(x) ,r为g(x)中最高次幂

  2. 转换数据:
    原始数据: 101001 转换为   K(x)= x^5+ x^3 +1(x^0)
    约定好的除数: 1101 转换为   g(x)=x^3+ x^2 +1…, r=3 (最低位为0 , 则 不写 )

被除数: k(x) * x^3 =x^8 +x^6 + x^3= 101001000

或者r=3,  则在原始数据后添加3个0 , 得到101001 000

运算:

被除数101001000 / 除数1101 (模二/异或运算,  相同为0,不同为1),得到余数 帧检验序列 FCS= 001
r=3, 则帧检验序列  得到的余数也是3位

  1. 发送的数据: 原始数据+FCS: 101001001

  2. 检验: 发送的数据 / 约定好的除数 , 余数为0, 正确

作用:实现无比特差错传输

3 网络

局域网(以太网Ethernet)

  1. 原因: 未形成统一的局域网标准

  2. 结果: 局域网的数据链路层分为 逻辑链路控制子层LLC(逐渐消失) 和 媒体接入控制子层MAC

  3. 服务:不可靠交付; 虚拟局域网VLAN

  4. 通信方式:半双工通信,交替进行

  5. 网状结构:

     总线型广播式以太网通信,同一时间仅允许一台主机通信,占用资源.因而出现了 载波监听多点接入/碰撞检测协议 CSMA/CD
