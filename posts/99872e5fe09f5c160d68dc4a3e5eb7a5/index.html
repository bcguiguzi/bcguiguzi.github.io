Java基础——final关键字的使用

前言

今天小编来带领大家回顾一下java当中final关键字的使用。
final关键字的使用，从以下几个方面进行总结：

含义

final关键字代表最终的、不可改变的。

  • 1.final关键字可以用来修饰一个类
  • 2.final关键字可以用来修饰一个方法
  • 3.final关键字可以用来修饰一个局部变量
  • 4…final关键字可以用来修饰一个成员变量

final关键字修饰类

基本格式 

public final class 类名称{
     //...
}

含义

当一个类被final修饰了，那它就是最终类，不能再有任何的子类（太监类）

注意事项

如果一个类是final的，那么其中所有的成员方法都无法进行覆盖重写（因为没有儿子）_

public final class MyClass {
    public void method(){
        System.out.println("方法执行！");
    }
}

//MyClass类被final修饰，已经是最终类了，不能有子类
public class MyClass2 /*extends MyClass*/ {
}

final关键字修饰方法

当final关键字修饰一个方法的时候，这个方法就是最终方法，也就是说这个方法不能被覆盖重写

基本格式 

访问修饰符 final 返回值类型 方法名（参数列表）{
     //方法体
 }

注意事项

对于类、方法来说，abstract关键字和final关键字不可以同时使用
原因：abstract要求子类必须重写，final关键字修饰的方法子类不能覆盖重写，两者矛盾

public abstract class Fu {

    public final void method(){
        System.out.println("父类方法执行！");
    }

    public abstract /*final*/ void methodA();
}

public class Zi extends Fu {
    //错误写法！不能覆盖重写父类当中的final方法
//    @Override
//    public void method() {
//        System.out.println("子类覆盖重写父类的方法");
//    }

    @Override
    public void methodA() {
        System.out.println("子类覆盖重写父类的抽象方法");
    }
}

final关键字修饰局部变量

一旦使用final用来修饰局部变量，那么这个变量就不能进行改变，不能二次赋值，可以理解为， “一次赋值，终身不变”

注意事项

（1）对于基本类型来说，不可变说的是变量当中的数据不可改变
（2）对于引用类型来说，不可变说的是变量当中的地址值不可改变
下面的例子当中，对于final修饰的Student对象内部成员变量进行重新赋值，不会影响对象的地址值，所以是可以给成员变量重新赋值的，但是当给创建的Student类对象赋值为一个新的Student对象stu2的的时候，就会报错，因为final修饰的对象地址值不可改变。

public class Demo03 {
    public static void main(String[] args) {

        //没用final修饰的话可以给变量重新赋值
        int num=10;
        System.out.println(num);    //10
        num=20;
        System.out.println(num);   //20

        //一旦使用final用来修饰局部变量，那么这个变量就不能进行改变，不能二次赋值
        final int num2=30;
        System.out.println(num2);       //30
//        num2=30;  代码报错
//        num2=40;  代码报错

        //正确写法！只要保证有唯一一次赋值即可
        final int num3;
        num3=30;

        //对于基本类型来说，不可变说的是变量当中的数据不可改变
        //对于引用类型来说，不可变说的是变量当中的地址值不可改变
        final Student stu2=new Student("高圆圆");
        System.out.println(stu2.getName());
        stu2.setName("高圆圆圆圆圆");
        System.out.println(stu2.getName());
        //错误写法，final的引用类型变量，其中的地址值不可以改变
//        stu2=new Student("赵又廷");

    }
}

final关键字修饰成员变量

对于成员变量来说，如果使用final关键字修饰，那么这个变量是不可变的，不可以重新赋值

注意事项

1.由于成员变量具有默认值，所以使用了final之后必须手动赋值，不会再给默认值了，因为默认值没有意义
2.对于final的成员变量，要么直接赋值，要么通过构造方法赋值，二者选其一。
3.通过构造方法给final成员变量赋值，必须保证类当中所有重载的构造方法都对final修饰的成员变量赋值
​

public class Person {
    //1.直接给final修饰的成员变量赋值
    private final String name="张三";
    private final int age;
    
    //2.用构造方法赋值,所有构造方法都要对final的成员变量进行赋值
    public Person() {
        age=30;
    }

    public Person(int age) {
        this.age=age;
    }
}
