qml踩坑系列——Scrollview和mouseArea冲突

问题描述

编写多任务视图的过程中，滚动区域（ScrollView）需要使用鼠标事件，但是增加上鼠标事件后，滚动区域就不能滚动了。

原因分析：

因为增加了mouseArea()，滚动区域的大小是根据内部子组件的大小进行扩展的，如果超出滚动区域的宽度或高度，就会触发滚动功能，但是增加了mouseArea（）,子控件的大小，并不会超过滚动区域，故不能滚动了

解决方案：

手动控制滚动区域的内容大小，不在让他自适应，如下：

  ScrollView{
        id : scrollView

        x : 0
        y : 0
        contentWidth : 10000
        contentHeight :10000
        width : 1080
        height: 1920
        MouseArea{
                anchors.fill : parent
            }
}    
控制contentWidth 或者contentHeight 大于width或者height 就可以即滚动，又可以使用鼠标事件


contentWidth：此属性保存内容宽度。
width：组件本身的宽度
